# 远程设备系统设置及仪表盘重构设计文档

## 1. 概述
在现有的 Flask 化工厂危险行为检测系统中，通过优化系统设置页面的布局来提高美观度，并集成实时的远程设备信息仪表盘。

## 2. 核心需求
1. **重构系统设置页面布局**：保留现有连接和指令下发功能，使 UI 更现代化、结构更清晰。
2. **新增设备信息仪表盘**：在连接 MQTT 成功后，订阅特定主题，实时解析并展示从边缘设备发来的运行状态与摄像头参数。

## 3. 设计方案 (方案 A - 卡片网格布局扩展)

### 3.1 前端 UI 布局
页面垂直分为三个逻辑层级：
1. **MQTT 连接管理区**：
   - 包含连接状态指示器和表单，可折叠或采用左右两栏（左边表单，右边历史记录）。
2. **边缘设备实时仪表盘区**（仅在连接成功时显示）：
   - 将各个 Camera 的 JSON 数据以数据卡片（Grid）形式排列。
   - 每张卡片显示：ID、位置、分辨率与帧率、当前缩放比例以及阈值信息。
   - 数据每秒或收到消息时通过 JavaScript 轮询动态更新。
3. **摄像头控制下发区**：
   - 保留原有的参数下发输入框和按钮。

### 3.2 后端数据流设计
1. **MQTT 订阅更新 (`mqtt_manager.py`)**：
   - 增加 `on_message` 回调函数。
   - 连接成功后，订阅主题 `jetson/info`。
   - 当接收到消息时，将 JSON 负载存储在 `MQTTManager` 类的属性中（例如 `self.latest_jetson_info`），自动忽略掉 `timestamp_ns`。
2. **新 API 端点 (`settings.py`)**：
   - 新增 `GET /settings/apijetson/info` 路由。
   - 读取并返回最新的 `latest_jetson_info` 数据，供前端 AJAX 轮询。

## 4. 实施步骤
1. 修改 `blueprints/mqtt_manager.py` 实现订阅和消息保存逻辑。
2. 修改 `blueprints/settings.py` 暴露 `/apijetson/info` 接口。
3. 重写 `templates/settings.html`：
   - 引入 Bootstrap 卡片和更具现代感的 CSS 样式。
   - 编写 JS 定时器（`setInterval`），连接成功时每隔 1~2 秒拉取最新仪表盘数据。
   - 动态渲染或更新对应的 DOM 节点。
