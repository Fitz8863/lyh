2. 技术路径选择

A

WebRTC WHIP (推荐)

优点： 毫秒级延迟，性能最强，直接对接 MediaMTX。
注意： 需要在浏览器开启 HTTP 麦克风白名单。

B

WebSocket 中转

优点： 后端可控，无需直接暴露 MediaMTX 给前端。
注意： 延迟略高于 WebRTC，实现复杂度较高。

推荐理由 (方案 A)

  • 架构最简洁：前端 -> MediaMTX -> 设备
  • MediaMTX 原生支持 WHIP，无需额外开发中转逻辑
  • 极低延迟，对讲体验最接近“打电话”

实现难点

  • 浏览器 HTTP 限制（需手动开启白名单）
  • 需要处理 WebRTC 的 ICE 打洞（局域网通常没问题）
